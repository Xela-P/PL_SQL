PL/SQL ORACLE - PARTIE 1 : Introduction

I.  INTRODUCTION

Le PL/SQL est le langage procédural d'ORACLE, c'est une extension du langage SQL qui est un langage ensembliste.

PL/SQL = Procédural Language / SQL

L'intérêt du PL/SQL est de pouvoir, dans un même traitement, allier la puissance des instructions SQL et la souplesse d'un langage procédural.

Le fonctionnement de PL/SQL est basé sur l'interprétation d'un "bloc" de commandes. Ce mode de fonctionnement permet d'obtenir des gains de transmission et des gains de performances.

Dans l'environnement SQL, les ordres du langage sont transmis et exécutés les uns à la suite des autres.

Dans l'environnement PL/SQL, les ordres SQL et PL/SQL sont groupés en BLOCS. Un bloc ne demande qu'un seul transfert et une seul exécution de l'ensemble des commandes contenues dans le bloc.

II.  LE BLOC PL/SQL

PL/SQL n'interprète pas une commande, mais un ensemble de commandes contenu dans un programme ou "bloc" PL/SQL.

Un bloc est composé de trois sections :

Chaque instruction de n'importe quelle section doit se terminer par un;. Possibilité de placer des commentaires :

  • -- Commentaire sur une ligne
  • /* Commentaire sur plusieurs lignes */

EXEMPLE sous SQL*PLUS de Personnal d'Oracle 7

Créer un fichierPLSQL_EX1.sql

1. Écriture du programme PL/SQL dans le fichier nomméPLSQL_EX1.sql:

Base de données : empdept
Version avec commentaires 
DECLARE -- Début du programme
sal_emp number(7,2); -- Variable local au bloc
BEGIN
	/* Sélectionner le salaire de l''employé saisi au préalable dans SQL*PLUS (num_emp),

	SELECT sal into sal_emp FROM emp

	If sal_emp < 1000 Then
		UPDATE emp SET sal = sal * 1.1
		WHERE empno '&num_emp';
	end if;

	/* commit; */
END;

III.  DÉCLARATION DES VARIABLES

La partie déclarative dans le un bloc PL/SQL peut comporter trois types de déclarations.

Elle (la partie déclarative) est délimitée par les mots-clésDECLARE, qui spécifie le début, etBEGIN, qui signifie la fin de la déclaration et le début de la partie des commandes.

Les types de déclarations possibles dans cette partie sont les suivants :

  • Déclaration des variables et des constantes
  • Déclaration des curseurs
  • Déclaration des exceptions

Les types de variables utilisés en PL/SQL sont les suivantes :

  • variables locales
    • -> de type Oracle : reconnu par Oracle Voir
    • -> faisant référence au dictionnaire de données voir
  • variables d'environnement extérieur à PL/SQL
    • -> champs d'écran de Forms
    • -> variables hôtes définies en langage hôte dans PRO*
    • -> variables définies dans SQL*PLUS (préfixées de&)

La déclaration d'une variable consiste à allouer un espace pour stocker et modifier une valeur. Elle est typée et peut recevoir une valeur par défaut et/ou un statutNOT NULL.

1.  Variables ou constantes locales de type Oracle et PL/SQL

Syntaxe :nom_variable [CONSTANT] type [[NOT NULL] := expression];

type:CHAR(n),NUMBER(n[,m]),DATEetBOOLEAN

expression: peut être une constante ou un calcul faisant éventuellement référence à une variable précédemment déclarée.

La déclaration d'une variable consiste à allouer un espace pour stocker et modifier une valeur. Elle est typée et peut recevoir une valeur par défaut et/ou un statutNOT NULL.

Exemple
DECLARE
	nom_du_client CHAR(10);
	X NUMBER / + 1; -- Initialisation
	PI constant NUMBER(7,5) := 3.14159;
	Rayon NUMBER := 1;
	Surface NUMBER(15,5) := PI * Rayon ** 2;
	Reponse BOOLEAN;

2.  Variables faisant référence au même type qu'une colonne d'une table ou même type qu'une autre variable

Syntaxe :nom_variable nom_table.nom_colonne%type;ounom_variable1 nom_variable2%type;

Exemple
DECLARE
	emp_nom emp.EMPNO%type; -- Même type que la colonne "EMPNO"
	X NUMBER(10,3);
	Y X%type; -- Du même type que X donc NUMBER(10,3)

Ceci est intéressant pour des raisons de simplification d'écriture et d'évolution.

IV.  VALORISATION DES VARIABLES PL/SQL

Trois possibilités de valorisation de variables sont disponibles :

  1. Par l'opérateur d'affectation ->:=
  2. Par la clause ->SELECT ... INTO ...
  3. Par le traitement d'uncurseurdans la sectionBEGIN

a.  Affectation de valeur

Syntaxe :nom_variable := expression;

expressionpeut être :

  • Une constante
  • Une variable
  • Un calcul -> opérateurs :+,-,*,/,**et||

Exemple
BEGIN
	X := 0:
	vnom := 'Monsieur'|| vnom; -- || = Concaténation
	Y := (X+5) * Y;

b.  La clause select ... into

La difficulté dans l'utilisation de la clauseselectrésulte du nombre de lignes ou d'occurences retournés.

Si leselectretourne une et une seule ligne, l'affectation s'effecture correctement.

Par contre,

  • Si leselectretourne 0 ligne : NO_DATA_FOUND(testnom_variable IS NULL), une erreur PL/SQL est générée
  • Si leselectretourne plusieurs lignes : TOO_MANY_ROWS, une erreur PL/SQL est générée

Syntaxe :SELECT <liste expression> INTO <liste variables> FROM <nom_table1>, <nom_tableN> ;

Exemple
DECLARE
	vref CHAR(10);
	vprif articles.PRIX%type;
	clt clients%rowtype;
BEGIN
	SELECT refart, prixart INTO vref, vprix
	FROM articles
	WHERE desart = 'Cadeau';

	SELECT * INTO clt
	FROM clients
	WHERE noclt = 10;
END ;

V.  STRUCTURES DE CONTRÔLES

a.  Structure alternative

Exemple 1 
if <condition> then
	[instructions1];
else [instructions2];
end if;
Exemple 2 
if <condition1> then
	[instructions1];
elsif <condition2> then
	[instructions2];
else [instructions3];
end if;

Seules les clausesif,thenetend ifsont obligatoires.

La condition peut utiliser les variables définies ainsi que tous les opérateurs présents dans SQL, à savoir :

  • =
  • <
  • >
  • <=
  • >=
  • <>
  • IS NULL
  • IS NOT NULL
  • BETWEEN
  • NOT BETWEEN
  • LIKE
  • NOT LIKE
  • AND
  • OR
  • OR NOT
  • IN
  • NOT IN

b.  La boucle POUR

Exemple 
for compteur in nb_debut..nb_fin
loop
	...
	[instructions];
	...
end loop;

Règles à savoir :

  • Déclaration caché de la variablecompteur
  • nb_debutetnb_finsont des constantes, expressions ou variables
  • compteurest une variable de type entier, locale à la boucle. Elle s'incrémente de 1 après chaque traitement du contenu de la boucle, jusqu'à ce qu'il atteigne la valeur de droite

c.  La boucle TANT QUE

Exemple 
while <condition>
loop
	...
	[instructions];
	...
end loop;

Laconditionest une expression définie en combinant les opérateurs SQL.

VI.  ÉCHANGES AVEC L'EXTÉRIEUR

A priori, il n'existe pas d'instruction d'affichage et de saisie dans le langage PL/SQL.

Sous Oracle :

  • On peut définir une variable (réservation d'une zone mémoire), et l'afficher à la fin du programme PL/SQL

    PLSQL_EX2.sql
    BEGIN
	SELECT COUNT(*) INTO :X FROM DEPT;
END;
/
    SQL> 
    variable x number
    SQL> 
    @ C:\path\to\file\PLSQL_EX2.sql
    SQL> 
    print x

    Remarque :Xest préfixée par:-> variable hôte.

  • On peut définir une valeur à rechercher (ou dans le fichier avantDECLARE)

    PLSQL_EX3.sql
    PROMPT "Nom du departement desire"
ACCEPT dept_nom

DECLARE
	res dept/rowtype; -- Même type qu'une ligne de la table
BEGIN
	SELECT * INTO res FROM dept WHERE dname = '&dept_nom';
END;
/
    SQL> 
    @ C:\path\to\file\PLSQL_EX3.sql

    Mais le résultat de la requête ne s'affiche pas.

    On peut définir une constante dans la console ou le fichier :

    SQL> 
    define dept_nom = 'SALES'
    SQL> 
    @ C:\path\to\file\PLSQL_EX3.sql

    Mais rien ne s'affiche.

    La meilleur solution pour récupérer le résultat de l'exécution d'un programme PL/SQL consiste à créer une table nomméresultatcomportant les champs que l'on désire, puis d'afficher à la fin du programme cette table.

    PLSQL_EX4.sql
    PROMPT "Quel est le departement desire"
ACCEPT numero

CREATE TABLE resultat (num NUMBER(2), nom CHAR(14))
/

DECLARE
	-- Syntaxe : nom_variable nom_table.nom_colonne%type
	numero dept.deptno%type;
	nomdept dept.deptname%type;
BEGIN
	SELECT deptno, dname INTO numero, nomdept
	FROM dept
	WHERE deptno = '&numero';

	INSERT INTO resultat VALUES (numero, nomdept);
END;
/

select * from resultat
/
drop table resultat
/
    SQL> 
    @ C:\path\to\file\PLSQL_EX4.sql

Remarque : dans un précompilateur Oracle, les variables sont désormais partagées avec le bloc PL/SQL. Ce bloc est insérer entre 2 délimiteursEXEC SQL EXECUTEetEND-EXEC.

VII.  UTILISATION DU PACKAGE DBMS OUTPUT

Sous Oracle, la packageDBMS_OUTPUTpermet d'afficher des messages à l'écran dans des programmes PL/SQL. Cela va facilité le test et le "débuggage" des programmes.

Pour cela il faut autoriser l'utilisation de l'instruction d'affichage :

SQL> 
set serveroutput on

ou

SQL> 
set serveroutput on size 80000

size 80000 précise le nombre de caractères maximum à afficher.

Dans un programme PL/SQL on peut alors utiliser l'instruction :

SQL> 
dbms_output.put_line('Message...');
PLSQL_EX4b.sql
PROMPT "Quel est le departement desire"
ACCEPT numero

DECLARE
	numero dept.deptno%type;
	nomdept dept.deptname%type;
BEGIN
	SELECT deptno, dname INTO numero, nomdept
	FROM dept
	WHERE deptno = '&numero';

	dbms_output.put_line('Le departement '|| TO_CHAR(numero) || ' a pour nom '|| nomdept);
END;
/
SQL> 
@ C:\path\to\file\PLSQL_EX4b.sql

Remarque : pour ne plus autoriser l'affichage :

SQL> 
set serveroutput off

Exercice 1 :Exo1_plsql.sql

Écrire le programme PL/SQL suivant :

  • L'utilisateur saisit le nom d'un employé
  • Si cet employé n'a pas de travail défini, vous devez afficher le nom de l'employé suivi den'a pas de travail
  • Si cet employé est un vendeur (SALESMAN), vous devez lui attribuer 1000 francs de commission et afficher le nom de l'employé suivi dea 1000 frs de commission
  • Si cet employé est dans la table mais n'est pas vendeur, vous devez lui affecter 0 de commission et afficher le nom de l'employé suivi den'a pas de commission

Vérifier si les modifications ont bien été effectués dans la table.

Exo1_plsql.sql
-- Activation du package dbms_output
set serveroutput on

PROMPT "Quel est le nom de l'employé ?"
ACCEPT nomemp

DECLARE
	jobemp emp.job%type;
	message char(30);
BEGIN
	SELECT job INTO jobemp
	FROM emp
	WHERE ename = '&nomemp';

	/* Si l'employé n'a pas de travail */
	if jobemp is null then
		message := '&nomemp'||' n''a pas de travail';
	end if;

	/* Si l'employé est un vendeur (SALESMAN) */
	if jobemp = 'SALESMAN' then
		UPDATE emp
		SET comm = 1000
		WHERE ename = '&nomemp';
		message := '&nomemp'||' a 1000 francs de commission';
	end if;

	/* Si l'employé est dans la table mais n'est pas vendeur */
	if jobemp is not null and jobemp != 'SALESMAN' then
		UPDATE emp
		SET comm = 0
		WHERE ename = '&nomemp';
		message := '&nomemp'||' n''a pas de commission';
	end if;

	/* Affichage du message grace au packages dbms_output */
	dbms_output.put_line(message);
END;
/
SQL>
